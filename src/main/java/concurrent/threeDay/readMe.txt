守护线程

1守护线程：通过设置Thread.setDaemon(true),同时在线程的start()开始前设置，否则报错；
2、守护线程中不要进行业务逻辑操作；
3、在守护线程中产生的线程也是守护线程；

线程阻塞
1、状态：（1）Thread.sleep();
                 （2）当线程wait()语句时，他会一直阻塞到通知（notify()）、被中断、或者经过制定毫秒时间为止。
                 （3）线程阻塞与各种不同I/O方式；常见的方式InputStream的read()方法；
                 （4）线程可以阻塞等待获取某个对象锁的排他性访问权限
                 
                 
注意：并非所有的线程阻塞都是可以被中断的，前2种是可以的，后2种是不可以的。
